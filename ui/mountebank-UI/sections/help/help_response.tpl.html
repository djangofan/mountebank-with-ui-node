Help for Response Injection

Fields for response. See http://www.mbtest.org/docs/api/injection for more information on using requests and state.
Sample function
    function (request, state, logger) {
    logger.info('origin called');
    state.requests = state.requests || 0;
    state.requests += 1;
    return {
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ count: state.requests })
    };
}

Response Stucture

Field Type Default
statusCode int 200
headers object { "Connection": "close" }
body string ""
_mode string - binary or text text

Request Structure

Field Description Type
requestFrom The client socket, primarily used for logging and debugging. string
path The path of the request, without the querystring string
query The querystring of the request object (as key value pairs)
method The request method string
headers The HTTP headers object (as key value pairs)
body The request body string

Note that the Mountebank server must run with the --allowInjection flag. Mountebank Injection documentation
